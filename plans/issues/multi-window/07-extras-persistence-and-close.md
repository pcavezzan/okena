## What to build

Make extra windows survive a quit/relaunch cycle when they were open at quit, and remove their entry from persistence the moment the user closes them. Keep main's "always present, closing main quits the app" semantics intact.

Behavior:

- **Save**: existing debounced auto-save and quit-time flush already serialize `WorkspaceData`. Since extras live inside `WorkspaceData.extra_windows` (slice 01), they're already persisted by every save. This slice ensures the OS bounds and any other transient-but-persisted fields (folder filter, collapsed map, hidden set, widths) write back into the right `WindowState` as the user resizes / interacts. Most of this is wired by the slice 02 mutation API; verify nothing slipped.
- **Restore at launch**: in `main.rs`, after loading `WorkspaceData` and creating the main OS window, iterate `extra_windows` and open one OS window per entry, each hosting a `WindowView` for its `WindowId`, with `os_bounds` honored.
- **Close extra**: when a user closes an extra window's OS window, remove that entry from `extra_windows` and drop the corresponding `WindowView` from `Okena::extra_windows`. This requires hooking the GPUI window-close callback for extras (not main) and dispatching the workspace mutation.
- **Close main**: closes the OS window per existing `LastWindowClosed` quit mode. Quit path flushes the workspace, so extras open at the moment of quit are saved exactly as they are. No special main-detection beyond what the OS / GPUI already provides.
- **Bootstrap on first launch / corrupt file**: covered by slice 01's invariant ŌĆö `main_window` always present, `extra_windows: vec![]` if missing/corrupt. This slice just relies on it.

After this slice: open the app, spawn W2, configure it (filter, hide a project, resize), quit. Relaunch ŌĆö main + W2 both come back with their state intact. Close W2 (by clicking its OS close button) ŌĆö the next launch shows main only, and W2's config is gone.

## Acceptance criteria

- [x] At launch, `main.rs` opens main's OS window first, then opens one OS window per `extra_windows` entry in order. Each new window hosts a `WindowView` keyed to its `WindowId`. `Okena::extra_windows` map is populated.
- [x] `os_bounds` from each persisted `WindowState` is honored on restore. If absent (e.g. just-spawned extra that hasn't received a resize yet), use cascade-offset from main.
- [x] Closing an extra OS window: `WorkspaceData.extra_windows` loses the entry, `Okena::extra_windows` loses the entity, save fires.
- [ ] Closing main: app quits per existing `LastWindowClosed`. Save flush captures everything still in `extra_windows`.
- [ ] After quit + relaunch, all extras that were open at quit reappear with the correct filter state, hidden set, widths, collapsed folders, and bounds.
- [ ] After quit + relaunch where the user had already closed an extra: that extra does NOT reappear.
- [x] Window resize updates `os_bounds` in the corresponding `WindowState` (debounced via the existing window-bounds observer pattern). Verify by resizing, quitting, relaunching ŌĆö bounds match.
- [x] First launch on a system with no `workspace.json`: app opens exactly one window (main), `extra_windows` empty.
- [x] Corrupt/missing `windows` section: app falls back to one fresh main window ŌĆö no panics.
- [ ] `cargo build` and `cargo test` both green.

## Notes

- GPUI window close callback: `cx.on_window_should_close` or the equivalent. For an extra, intercept and remove from `WorkspaceData` before the actual close. For main, do nothing extra ŌĆö `LastWindowClosed` handles quit and the existing `cx.on_app_quit` saves state.
- Be careful with the order of operations on close: drop the `WindowView` entity AFTER the workspace mutation, not before, otherwise observers may fire on a half-deleted state.
- The PTY event loop and other shared infrastructure already iterate the live windows via `Okena::extra_windows.values()` after slice 03. Verify that closing an extra doesn't leave dangling observers.
- Save behavior for transient WindowState fields (folder filter, hidden, widths, collapsed, bounds) was wired in slices 02 and 05; this slice does not redo it, only verifies round-trips work for the full lifecycle.
